## 一、系统边界与数据流全景

### 1.1 系统关系

```
┌─────────────────────────────────────────────────────────┐
│  V2 系统（已有）                                         │
│  - AI 录单解析                                          │
│  - 运单主数据（结构化运单）                                │
│  - 对外暴露 HTTP API                                     │
└────────────┬────────────────────────────────────────────┘
             │ HTTP API（鉴权）
             ▼
┌─────────────────────────────────────────────────────────┐
│  V3 系统（本次构建）                                      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐ │
│  │ 扫描品控  │  │ 异常上报  │  │ 分级审批  │  │ 执行联动 │ │
│  │ 规则引擎  │  │ 手工录入  │  │ 状态机    │  │ 赔付/库存│ │
│  └──────────┘  └──────────┘  └──────────┘  └─────────┘ │
│                     ↓                                   │
│  ┌──────────────────────────────────────────────────┐  │
│  │  V3 独立数据库（Neon/Supabase/Turso）              │  │
│  │  - 运单本地快照  - 异常工单  - 审批记录            │  │
│  │  - 赔付记录      - 库存数据  - 扫描记录            │  │
│  │  - 品控规则      - 接口日志                        │  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 1.2 核心数据流向

| 阶段 | 数据来源 | 触发方式 | 数据去向 |
|------|---------|---------|---------|
| 运单同步 | V2 API | 按需/定时 | V3 运单快照表 |
| 扫描品控 | 手工输入SKU | 扫描操作 | 扫描记录表 → 品控规则引擎 → 异常工单 |
| 异常上报 | V2 API + 快照 | 手工上报 | 异常工单表 |
| 分级审批 | 异常工单 | 审批操作 | 审批记录表 → 联动赔付/库存 |
| 执行联动 | 审批记录 | 审批通过 | 赔付记录表 / 库存表 |

## 二、状态机详细分析

### 2.1 工单状态机（物流异常 / 品控异常共用审批流程）

```
                    ┌──────────────┐
                    │   待审批      │ ← 拒绝重提（次数上限内）
                    └──────┬───────┘
                           │ 审批人领取
                    ┌──────▼───────┐
  拒绝（返回待审批）←─│ 一级审批中    │──── 金额超阈值 ────┐
                    └──────┬───────┘                    │
                           │ 通过                      │
                    ┌──────▼───────┐          ┌────────▼────────┐
                    │   执行中      │          │  二级审批中      │
                    │ 理赔/发货/退货│          └────────┬────────┘
                    └──────┬───────┘                   │ 通过
                           │                   ┌───────▼────────┐
                    ┌──────▼───────┐          │     执行中       │
                    │   已完成      │          └───────┬────────┘
                    └──────────────┘                  │
                                               ┌──────▼────────┐
                                               │    已完成       │
                                               └───────────────┘

异常分支：
- 超时未处理 → 自动升级到下一级 / 自动驳回
- 拒绝次数超上限 → 自动关闭 / 自动升级
- 二级审批超时 → 自动驳回

品控特殊路径：
- 品控工单跳过"待审批"，直接进入二级审批（因为品控是系统自动检测触发的）
```

### 2.2 扫描批次状态机（品控专用，与工单状态机分离）

```
                    ┌──────────────┐
                    │   扫描录入    │
                    └──────┬───────┘
                           │ 品控规则引擎检测
                    ┌──────┴───────┐
                    │              │
              通过  │              │  异常
                    ▼              ▼
             ┌──────────┐   ┌──────────────┐
             │ 正常出库  │   │  品控暂扣     │
             │(流程结束) │   │ (批次锁定)    │
             └──────────┘   └──────┬───────┘
                                   │
                    ┌──────────────┼──────────────┐
                    │              │              │
              超时未处理      正常路径      品控主管误判
                    │              │              │
                    ▼              ▼              ▼
           强制升级二级审批   创建异常工单    快速放行
                              (来源=扫描触发)  (绕过审批)
                                   │              │
                                   ▼              ▼
                       ┌──────────────┐   ┌──────────────┐
                       │ 进入工单审批  │   │ 批次解锁     │
                       └──────────────┘   │ 留痕记录     │
                                          └──────────────┘
```

### 2.3 两个状态机的关联规则

| 规则 | 说明 |
|------|------|
| 关联方式 | 扫描记录表通过 `ticket_id` 关联工单表 |
| 批次锁定 | 品控暂扣期间，批次 SKU 不可被其他运单引用 |
| 幂等性 | 同一批次有未关闭工单时，重复扫描只追加记录，不新建工单 |
| 事务一致性 | 执行动作时必须同步更新扫描批次状态，两个变更在同一个事务内 |
| 解锁时机 | 仅在工单完成（正常/快速放行）时解锁批次 |

## 三、功能模块拆解

### 模块零：扫描操作与品控检测

| 功能点 | 类型 | 说明 |
|--------|------|------|
| SKU 扫描录入 | 必须 | 手工输入条码/SKU，不强制接硬件 |
| SKU 归属校验 | 必须 | 通过 V2 接口验证 SKU 属于真实运单 |
| 品控规则引擎 | 必须 | 根据可配置规则判定通过/异常 |
| 品控暂扣 | 必须 | 异常时锁定批次，禁止出库 |
| 自动创建工单 | 必须 | 异常时自动创建，来源标记"扫描触发" |
| 扫描幂等性 | 必须 | 同一批次有未关闭工单，不重复创建 |
| 误判快速放行 | 必须 | 仅品控主管可操作，需留痕 |
| AI 辅助判断 | 可选 | 大模型推荐异常类型/严重度 |

### 模块一：异常工单上报

| 功能点 | 类型 | 说明 |
|--------|------|------|
| 运单真实性校验 | 必须 | 调用 V2 接口实时校验（非仅本地快照） |
| 归属校验 | 必须 | 上报人只能对自己的运单上报（多租户隔离） |
| 重复上报阻断 | 必须 | 同类型未关闭工单不允许重复上报 |
| 异常类型选择 | 必须 | 物流类：丢件/破损/拒收/超时/地址错误 |
| AI 辅助分类 | 可选 | 大模型推荐异常类型，可人工修改 |

### 模块二：分级审批流程引擎

| 功能点 | 类型 | 说明 |
|--------|------|------|
| 审批流转 | 必须 | 完整实现一级/二级审批状态机 |
| 分级规则可配置 | 必须 | 金额阈值可在后台调整，不硬编码 |
| 并发冲突 | 必须 | 两人同时审批，只有一人操作生效 |
| 审批人兜底 | 必须 | 审批人禁用后有转交/上报机制 |
| 超时自动流转 | 必须 | 超时自动升级/驳回，需后台任务 |
| 权限边界 | 必须 | 上报人不能审批自己的工单 |
| 幂等性 | 必须 | 重复点击不产生重复记录 |

### 模块三：执行联动

| 功能点 | 类型 | 说明 |
|--------|------|------|
| 理赔流程 | 必须 | 生成赔付记录 |
| 重新发货 | 必须 | 联动库存扣减 |
| 退货入库 | 必须 | 联动库存增加 |
| 事务一致性 | 必须 | 审批状态变更与下游联动在同一事务 |
| 可追溯性 | 必须 | 赔付/库存变更可反查审批记录 |
| 品控执行动作 | 必须 | 放行/退供应商/重采购/降级处理 |
| 赔付方向区分 | 必须 | 品控=追偿供应商，物流=赔付客户 |

### 模块四：工单列表与追踪

| 功能点 | 类型 | 说明 |
|--------|------|------|
| 列表筛选 | 必须 | 按状态/异常类型/运单号/审批人筛选 |
| 分页 | 必须 | 大数据量下流畅展示 |
| 详情页 | 必须 | 完整状态变更历史 + 审批意见 |
| 超时预警 | 必须 | 即将超时工单高亮提示 |
| 模拟数据 | 必须 | 至少 200 条工单 |

### 模块五：跨系统接口与数据一致性

| 功能点 | 类型 | 说明 |
|--------|------|------|
| 接口监控页 | 必须 | 同步时间/成功率/调用日志 |
| 数据来源标注 | 必须 | 实时 vs 缓存明确标注 |
| V2 不可用降级 | 必须 | 有清晰降级提示，不自屏崩溃 |
| 接口鉴权 | 必须 | API Key/Token |
| 重试机制 | 必须 | 超时 + 1-2 次重试 |
| Request ID 追踪 | 必须 | 每次调用生成可追踪 ID |

### 模块六：假设说明文档（强制交付）

| 序号 | 留白项 | 状态 |
|------|--------|------|
| ① | 分级审批金额阈值 | 需补全 |
| ② | 审批超时时长 | 需补全 |
| ③ | 重提次数上限 | 需补全 |
| ④ | 物流异常类型映射 | 需补全 |
| ⑤ | 角色权限划分 | 需补全 |
| ⑥ | V2 数据同步策略 | 需补全 |
| ⑦ | 品控暂扣超时时长 | 需补全 |
| ⑧ | 品控规则触发阈值 | 需补全 |
| ⑨ | 品控主管权限边界 | 需补全 |

## 四、技术难点梳理

### 4.1 高优先级难点

| 难点 | 影响范围 | 风险 |
|------|---------|------|
| 两套状态机分离但关联 | 整个审批流程 | 批次解锁/工单关闭不一致 |
| 审批-库存-赔付事务一致性 | 模块三 | 中间态数据（已通过但库存未联动） |
| 跨系统数据一致性 | 模块五 | V2 更新后 V3 快照过期 |
| 并发审批冲突 | 模块二 | 同一工单两条矛盾审批记录 |

### 4.2 中优先级难点

| 难点 | 影响范围 | 风险 |
|------|---------|------|
| 审批人兜底机制 | 模块二 | 工单永久卡死 |
| 超时自动流转 | 模块二 | 后台任务设计、定时扫描性能 |
| 归属校验（多租户） | 模块一 | 跨租户数据泄露 |
| V2 不可用降级 | 模块五 | 整体系统崩溃 |

### 4.3 低优先级难点

| 难点 | 影响范围 | 风险 |
|------|---------|------|
| 品控规则可配置 | 模块零 | 规则引擎设计复杂度 |
| 扫描幂等性 | 模块零 | 重复创建工单 |
| 200 条模拟数据 | 模块四 | 种子脚本设计 |
