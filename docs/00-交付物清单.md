## 已实现页面路由

| 路由 | 页面 | 类型 |
|------|------|------|
| `/` | 首页仪表盘 | 静态 |
| `/tickets` | 工单列表（含筛选/分页） | 静态 |
| `/tickets/[id]` | 工单详情（审批操作/审批历史/扫描记录/赔付记录） | 动态 |
| `/tickets/create` | 异常上报（运单校验+类型选择） | 静态 |
| `/scan` | 扫描品控（录入+规则检测+结果展示） | 静态 |
| `/approval` | 审批管理（待审批列表+操作） | 静态 |
| `/admin/rules` | 品控规则配置（可配置项展示） | 静态 |
| `/admin/monitor` | 接口监控（同步日志+统计） | 静态 |
| `/api/cron/check-timeout` | 超时检测定时任务 | 动态 |

## 待接入（后续步骤）

1. **连接数据库**: 配置 `.env` 中的 `DATABASE_URL`（Supabase/Neon）
2. **运行 Drizzle 迁移**: `npx drizzle-kit push`
3. **运行种子数据**: `npx tsx src/lib/db/seed.ts`
4. **配置 V2 接口**: 设置 `V2_API_BASE_URL` 和 `V2_API_KEY`
5. **部署 Vercel**: 关联 Git 仓库自动部署
6. **接入大模型**（可选加分）：配置 `OPENAI_API_KEY`
